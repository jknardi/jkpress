# Contributing to JkPress

We love your input! We want to make contributing to JkPress as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## English Version

### Development Process
We use GitHub to host code, to track issues and feature requests, as well as accept pull requests.

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.
6. Issue that pull request!

### Any contributions you make will be under the MIT Software License
When you submit code changes, your submissions are understood to be under the same [MIT License](http://choosealicense.com/licenses/mit/) that covers the project. Feel free to contact the maintainers if that's a concern.

### Report bugs using Github's [issues](https://github.com/jknardi/jkpress/issues)
We use GitHub issues to track public bugs. Report a bug by [opening a new issue](https://github.com/jknardi/jkpress/issues/new); it's that easy!

**Great Bug Reports** tend to have:

- A quick summary and/or background
- Steps to reproduce
  - Be specific!
  - Give sample code if you can.
- What you expected would happen
- What actually happens
- Notes (possibly including why you think this might be happening, or stuff you tried that didn't work)

### Use a Consistent Coding Style

* 2 spaces for indentation rather than tabs
* You can try running `npm run lint` for style unification

### License
By contributing, you agree that your contributions will be licensed under its MIT License.

## Versi Bahasa Indonesia

### Proses Pengembangan
Kami menggunakan GitHub untuk hosting kode, melacak isu dan permintaan fitur, serta menerima pull request.

1. Fork repositori dan buat branch Anda dari `main`.
2. Jika Anda menambahkan kode yang harus diuji, tambahkan tes.
3. Jika Anda mengubah API, perbarui dokumentasi.
4. Pastikan semua tes lolos.
5. Pastikan kode Anda sesuai dengan standar (lint).
6. Kirim pull request!

### Setiap kontribusi yang Anda berikan akan berada di bawah Lisensi MIT
Saat Anda mengirimkan perubahan kode, kontribusi Anda dianggap berada di bawah [Lisensi MIT](http://choosealicense.com/licenses/mit/) yang sama yang mencakup proyek ini. Jangan ragu untuk menghubungi pengelola jika ini menjadi perhatian.

### Laporkan bug menggunakan [issues](https://github.com/jknardi/jkpress/issues) GitHub
Kami menggunakan isu GitHub untuk melacak bug publik. Laporkan bug dengan [membuka isu baru](https://github.com/jknardi/jkpress/issues/new); semudah itu!

**Laporan Bug yang Baik** biasanya memiliki:

- Ringkasan singkat dan/atau latar belakang
- Langkah-langkah untuk mereproduksi
  - Bersikaplah spesifik!
  - Berikan contoh kode jika bisa.
- Apa yang Anda harapkan akan terjadi
- Apa yang sebenarnya terjadi
- Catatan (mungkin termasuk mengapa Anda pikir ini mungkin terjadi, atau hal-hal yang sudah Anda coba tetapi tidak berhasil)

### Gunakan Gaya Koding yang Konsisten

* 2 spasi untuk indentasi daripada tab
* Anda dapat mencoba menjalankan `npm run lint` untuk penyatuan gaya

### Lisensi
Dengan berkontribusi, Anda menyetujui bahwa kontribusi Anda akan dilisensikan di bawah Lisensi MIT.
